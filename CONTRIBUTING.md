# Contributing a Skill

Anyone with collaborator access to this repo can add or update skills. Here's how.

---

## The quick version

1. Create or edit a `SKILL.md` file in the right folder
2. Commit and push (or submit a pull request)
3. Done — the skill appears in Cowork's Browse plugins automatically

---

## Option A — GitHub web UI (no git required)

1. Go to [github.com/Cal1234567/claude-plugins](https://github.com/Cal1234567/claude-plugins)
2. Navigate to the right folder:
   - `plugins/investments/` — for investment, diligence, or client work skills
   - `plugins/productivity/` — for workflow, summaries, or general productivity skills
3. Click **Add file → Create new file**
4. Name it: `my-skill-plugin/skills/my-skill/SKILL.md`
5. Write the skill (see format below)
6. Click **Commit changes** — use a message like `add my-skill skill`

---

## Option B — CLI (for technical contributors)

From anywhere on your machine:

```powershell
# 1. Create the skill stub in your personal skills folder
.\scripts\new-skill.ps1 -Name "my-skill" -Description "Does X when user asks Y"

# 2. Edit the skill
# Open C:\Users\<you>\.claude\skills\my-skill\SKILL.md and fill it in

# 3. Promote to the repo
.\scripts\promote-skill.ps1 -Name "my-skill" -Category "Investments"

# 4. Push
git add . && git commit -m "add my-skill skill" && git push origin main
```

---

## SKILL.md format

Every skill is a single markdown file with a YAML frontmatter block followed by instructions.

```markdown
---
name: my-skill
description: >-
  One or two sentences describing what this skill does and when to use it.
  Be specific about the triggers — what the user says to invoke it.
---

# My Skill

Instructions for Claude go here. Be direct and precise — this is what Claude
reads every time the skill is invoked.

## What to do

1. Step one
2. Step two
3. Step three
```

**Tips for writing a good skill:**
- The `description` is what shows in Cowork's Browse plugins and what Claude uses to decide when to trigger the skill — make it specific
- The body is the actual instructions — write it like you're briefing a smart colleague
- One skill = one job. If it does three things, split it into three skills.
- Test it in Cowork before pushing — invoke it with `/my-skill` or describe the task naturally

## Folder structure

```
plugins/
  investments/
    my-skill-plugin/
      .claude-plugin/
        plugin.json        ← auto-generated by promote-skill.ps1
      skills/
        my-skill/
          SKILL.md         ← this is the file you write
  productivity/
    ...
```

If you're using the GitHub web UI, you only need to create `SKILL.md`. The `plugin.json` will be generated next time someone runs `promote-skill.ps1`, or you can copy one from an existing plugin and update the name/description.
